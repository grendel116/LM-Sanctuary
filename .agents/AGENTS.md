# Project Scoped Rules

## Core Engineering Principles
- **Root Cause Resolution**: Resolve structural errors directly at their source schemas, configs, and formats.
- **Simplicity**: Maintain direct, transparent control flows.
- **Minimal Footprint**: Reuse logic and keep files small.
- **UI Aesthetics**: Adhere strictly to predefined CSS tokens and class standards.
- **Affirmative Instructions Only**: Write guidelines, rules, and system prompts using positive, active directives. Express instructions by explaining what to do instead of what to avoid. If a negative directive is necessary for clarity, present it as a standalone instruction.
- **Vocabulary Constraints**: Use separate, single word tokens instead of hyphenated words. Select single, precise adjectives to describe concepts rather than stacking multiple consecutive adjectives.

## Dialectical Materialist Design Ethos
- **Anti Patchwork**: Build cohesive, structural systems rather than layering hotfixes.
- **Systemic Resolution**: Eliminate root contradictions to keep codebase free of software debt and alienation.
